INSTALL CUTPILOT

选一个 AI，
开始剪片。

当前版本支持 macOS。素材分析、工程和渲染默认留在本机。

准备下载连接 AI验证模型配置

准备这台 Mac

  • Node.js 18 或更新版本
  • Google Chrome
  • FFmpeg 与 FFprobe
  • ImageMagick（可选）

使用 Homebrew 安装媒体工具：

brew install ffmpeg imagemagick xcode-select --install

一键安装 CutPilot

git clone https://github.com/Hellotravisss/cutpilot.git cd cutpilot ./install-macos.sh --install-deps

安装后可完全独立启动：

~/.local/bin/cutpilot --project ~/Movies/my-video.cutpilot.json --media ~/Movies/clips

仓库目前公开可见，但尚未授予开源复用许可证。请勿在未获授权时重新分发代码。

连接你的 AI 主机

Codex添加本地 MCP 服务Claude Code运行随附安装脚本Claude Desktop运行安全配置脚本

Codex

把仓库内的 codex.mcp.json 作为配置参考，并将路径替换为本机 CutPilot 的绝对路径：

{ "mcpServers": { "cutpilot": { "command": "node", "args": ["/absolute/path/to/cutpilot/scripts/server.mjs"] } } }

Claude Code

chmod +x claude/install-claude-code.sh ./claude/install-claude-code.sh

Claude Desktop

脚本会备份现有配置，保留其他 MCP 服务，再加入 CutPilot。完成后重新启动 Claude Desktop。

chmod +x claude/install-claude-desktop.sh ./claude/install-claude-desktop.sh

确认安装成功

在 AI 对话中输入：

请列出 CutPilot 支持的项目模板，然后运行运行环境检查。不要修改任何文件。

看到项目模板与 FFmpeg、Chrome 等检查结果后，就可以告诉 AI 素材文件夹和目标视频类型。需要手工调整时，让 AI 打开本地 review 界面。

独立模型与生成服务

在编辑器的“AI”标签中可配置 OpenAI、Anthropic 或兼容接口。密钥只写入本机用户配置，不进入项目文件。配置后无需 Codex 或 Claude，也能直接对话生成并审核剪辑方案。

视频、音乐、音效和远程转录服务仍需要你自己的供应商账号、端点和计费。

不要把 API 密钥写进 GitHub、项目文件或聊天消息。所有 AI 时间线修改仍需人工确认。
